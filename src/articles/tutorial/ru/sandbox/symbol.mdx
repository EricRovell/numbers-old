# Символ

Первое, с чем необходимо познакомиться - символ. Символ - это краеугольный камень, на котором строятся числа в приложении.

Символ выполняет функцию условного обозначения некоторого числа. Привычные цифры 0, 1, 2... - символы, условно принятые за репрезентацию чисел. Таким же образом в роли символа для обозначения чисел могут быть использованы и буквы латинского алфавита. На самом деле, символ может даже являться физическим объектом. Если подумать, то монета определённого номинала играет роль числа. Это явно интереснее, нежели использование в качестве примеров камней или палочек.

В нашем случае воспользуемся лампочкой - как удобным объектом для моделирования чисел двоичной системы счисления. Лампочка включена - единица, выключена - нуль. Попробуем поработать с этой репрезентацией символа собственноручно:

<div style={{ padding: "1em 0", placeSelf: "center" }}>
  <Symbol
    mode="binary"
    skin="default"
    baseIn={2}
    labelsUp="none"
    labelsDown="none"
    locale={locale.digits} />
</div>

Таким образом мы будем взаимодействовать с одной или множеством символов-лампочек, образуя различные числа. Работать с такими числами мы начнём уже в следующей главе - <Link href="/[lang]/tutorial/[...route]" as="/ru/tutorial/ru/sandbox/symbols" title="Символы" label="Символы" />.

## Символ "по умолчанию"

Для работы с произвольными системами счисления по умолчанию используется панель с отображением привычного символа, имеющая более обширный функционал и вид. Для примера поработаем с интерактивным символом с системой счисления 16. Латинские буквы используются для репрезентации чисел после 9.

  <Symbol
    baseIn={16}
    locale={locale.digits} />
</div>

### Центральный блок: Значение

Центральный блок отображает текущее состояние символа, другими словами, число. Состояние можно изменить двумя способами: непосредственным вводом желаемого значения в блок, либо используя средние блоки для перехода на один шаг.

> Пользовательский ввод проходит проверку на "адекватность". Только доступные символы для данной системы счисления будут приняты на ввод, в противном случае символ даст об этом знать. Стоит попробовать.

> Исключением является режим двоичных чисел, пользовательский ввод заменён на переключение при взаимодействии с пользователем.

### Средние блоки: Шаг

Средние блоки позволяют изменять значение символа на один шаг. 

> Значения символа при использовании средних блоков зацикленны. Это значит, что при увеличении максимального значения символ перейдёт в минимальное состояние. И наоборот, в случае уменьшения минимального - в максимальное. 

### Крайние блоки: Подписи

Крайние блоки отображают вспомогальную информацию для пользователя - подписи.

Что это за вспомогальная информация и зачем она нужна? Дело в том, что для составления числа чаще необходимо несколько символов, в зависимости от самого числа, которое мы пытаемся представить, и степени системы счисления. Подробнее об этом можно почитать в главе про <Link href="/[lang]/tutorial/[...route]" as="/ru/tutorial/ru/bases/positional" title="Позиционная система счисления" label="позиционные системы счисления." />.

При использовании нескольких символов в большом количестве, либо на мобильной версии может быть полезна информация об индексе символа. В случае перевода чисел из одной системы в другую, информация об образующей степени или значении степени поможет лучше понять принцип перевода и упросить понимание. При необходимости подписи можно скрыть.

> При скрытом режиме подписей очередное взаимодействие вернёт их на место.

Переключение между типами подписей возможно касанием/кликом непосредственно по желаемому блоку. Итого, возможные состояния могут принимать значения:

- индекс символа;
- образующая степень;
- значение образующей степени / символа;
- отсуствие информации;

> Крайние подписи имеют независимое состояние в зависимости от расположения. Таким образом возможны два режима подписей одновременно.
