# Символы

Символами в данном приложении мы будем называть упорядоченное множество символов, используемых для записи числа. Другими словами, это привычное нам число, разбитое на "атомы", где каждый атом - самостоятельный символ, с которыми мы познакомились <Link href="/[lang]/manual/[...route]" as="/ru/manual/ru/sandbox/symbol" title="Символ" label="в прошлой главе" />.

> Если более подробно, то мы будет работать с позиционными системами счисления со значением степени от 2 до 36. По этой причине числа не всегда можно записать с помощью одного символа. Для более подробной информации советуется освежить главы <Link href="/[lang]/manual/[...route]" as="/ru/manual/ru/bases/numeral-bases" title="Системы счисления" label="по основам систем счисления" /> и <Link href="/[lang]/manual/[...route]" as="/ru/manual/ru/bases/positional" title="Позиционные системы счисления" label="по позиционным системам счисления" />.

## Анатомия символов

Символы, как элемент пользовательского взаимодействия, состоят из двух основных блоков: числового контейнера и панели управления. Если контейнер выполняет лишь техническую функцию для хранения символов вместе, то панель управления необходимо рассмотреть подробнее.

### Панель управления

Панель управления расположена непосредственно под символами и позволяет контролировать такие важные параметры как степень системы счисления и количество имеющихся в системе символов. Внешним видом панель напоминает символ, повёрнутый набок.

Функциональность центрального блока аналогична центральному блоку символа, с единственным лишь отличием, что в нём фиксируется значение степени системы счисления. Средние блоки повторяют функционал средних блоков символа и предназначены для изменения степени с шагом на единицу.

> Пользовательский ввод так же проходит валидацию на "адекватность" и внутренние ограничения приложения. Поддерживаются лишь системы счисления от 2 до 36 включительно. Переключение значений с помощью среднего блока зацикленно.

Крайние блоки как элемент управления позволяют контроллировать количество символов. 

> Диапазон допустимого количества символов является неизменяемым и на момент написания статьи позволяет использовать от 0 до 16 символов включительно. Однако, пользователь может указать свои ограничения <Link href="/[lang]/settings" as="/ru/settings" title="Пользовательские настройки" label="в разделе настроек" /> с учётом того, что пользовательские значения не выходят за пределы данных значений.

Панель управления неразрывно связана с __символами__, объединяя их вместе, создавая цельное число, разбитое _визуально_ для удобства пользователя.
